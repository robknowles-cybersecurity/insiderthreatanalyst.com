Emerging Threat Vectors

Insider threat detection was built around network perimeters and office-based work. These vectors exploit the gaps that remote work, 5G, and commercial proxy services have opened. Your existing detections probably don't cover them.

Threat Landscape at a Glance

DPRK IT workers
Stolen identities for remote jobs. TA integration, identity verification, behavioral + technical indicators. Google GTIG · Nisos
Overemployment
Undisclosed multiple positions. COI, IP exposure, performance degradation. This is where the practitioners are.
Work avoidance
Mouse movers, keyboard weights, IP-KVMs, keepawake scripts, solo Zoom. Detection & case lifecycle.
AI-powered threats
Deepfakes, LLM-assisted exfil planning, AI pretexts. Uncharted governance territory.
Employment fraud
Fabricated credentials, hiring ID theft, contractor substitution. NBC/Nisos investigation (Mar 2026).
Smart glasses
Wearable exfiltration — live-streaming boardrooms, photographing screens without touching a device. LED indicators can be disabled. 7M+ sold 2025. Legal analysis (4-part)
Shadow AI
Unsanctioned AI tools processing sensitive data through personal accounts. Adoption outpacing governance. AI agents amplifying risk.
Insider recruitment
Threat actors actively recruiting employees via Telegram, Signal, dark web. 91K+ instances observed in 2025. Telecoms, finance top targets.

🌐 Residential Proxies K-WAVD01

Residential proxy services route traffic through real ISP IP addresses — Comcast, AT&T, Verizon home connections. The traffic looks like a normal residential user, because it is coming from a normal residential IP. This defeats geofencing, UEBA location baselines, and "impossible travel" alerts.

These aren't Tor or VPNs. Commercial residential proxy providers (Bright Data, Smartproxy, SOAX, IPRoyal) sell access to millions of real IPs, often harvested from free VPN apps or SDK monetization. The user appears to be browsing from any city, any state, any country — on demand.

Why this matters now: The FBI raided laptop farms in Arizona and other states where DPRK IT workers were using residential proxies to mask their true location while holding remote US jobs under stolen identities. This wasn't theoretical — it was operational, at scale, generating revenue for North Korea's weapons programs.
Insider threat use cases
CRIT
Nation-state IT worker fraud (DPRK)
Operatives use stolen US identities to secure remote IT jobs. Residential proxies mask their true location (often China, Russia, or SE Asia). Laptop farms in-country handle any hardware token or geofence checks. Revenue funds WMD programs. FBI/DOJ indictments in 2024-2025 confirm this at scale.
HIGH
Malicious exfiltration masking
An insider exfiltrating data routes traffic through residential proxies to appear as normal web browsing from expected locations. DLP that relies on destination IP reputation or geo-analysis is blind. The exfil channel looks like a user browsing from home.
MED
Overemployment evasion
Employees working multiple remote jobs simultaneously use proxies to mask activity patterns. Not malicious per se, but violates employment agreements and creates split-attention risk, IP contamination, and potential data commingling across employers.
What breaks
  • Geofencing / geo-restriction — proxy IP is in the "right" location
  • Impossible travel detection — no travel detected because all sessions appear local
  • UEBA location baselines — residential IPs pass reputation checks
  • IP-based threat intelligence — residential IPs aren't on blocklists
  • VPN/Tor detection — these aren't VPN or Tor exit nodes
📡 Rogue WiFi / Personal Hotspots / Network Bridging

An employee connects their corporate device to a personal phone hotspot instead of the corporate network. Every byte of traffic bypasses your firewall, proxy, DLP, IDS, and SIEM. The device is online, the user is working — your monitoring sees nothing.

5G makes this worse. Personal hotspots now deliver 100+ Mbps — fast enough that there's no performance penalty. The user has no reason to switch back to corporate WiFi, and no technical barrier to staying off-network indefinitely.

Three scenarios, escalating severity
LOW
Monitoring bypass (most common)
Employee connects to hotspot for personal browsing during work, or because corporate WiFi is slow. No malicious intent — just convenience. But your UEBA, web proxy, and network-based DLP are now blind to this user's activity.
HIGH
Data exfiltration channel
Insider deliberately uses hotspot to exfiltrate data outside all network monitoring. Files go straight from corporate device to personal cloud storage over cellular. Your DLP never sees the transfer. No proxy log, no firewall event, no CASB alert.
CRIT
Network bridging
A device connected to both corporate WiFi and a personal hotspot simultaneously bridges the two networks. This creates a backdoor from the internet into your corporate network — accessible from the parking lot, a nearby building, or anywhere the hotspot reaches. A compromised phone becomes an entry point.
Regulatory note: PCI DSS Requirement 11.1 already mandates quarterly rogue wireless access point scans. If your organization handles cardholder data, you're required to detect this. Most organizations doing rogue AP scans are only looking for unauthorized access points on premises — not employees' phone hotspots.

Detection approaches

Network-based detection is, by definition, blind to these vectors — the traffic never touches your network. Detection must move to the endpoint and to behavioral analysis that doesn't depend on network visibility.

Endpoint adapter monitoring
EDR/UEM agents that detect when a device connects to a non-corporate network, logs adapter changes, and flags hotspot SSIDs. CrowdStrike, SentinelOne, Microsoft Intune all support this.
WIPS / RF scanning
Wireless Intrusion Prevention Systems detect rogue access points and unauthorized wireless connections within your physical perimeter. Cisco Meraki, Aruba, Ekahau.
NAC with 802.1X
Network Access Control that enforces device posture before granting network access. Can detect and block devices with active secondary network interfaces. Cisco ISE, Aruba ClearPass, Forescout.
USB / peripheral controls
Disable USB tethering and mobile hotspot connectivity via MDM/UEM policy. Prevents the most common bridging vector. Microsoft Intune, JAMF, VMware Workspace ONE.
Behavioral session analysis
UEBA that keys on session characteristics (typing cadence, mouse behavior, application patterns) rather than IP/location. Detects anomalous users regardless of their network path. DTEX, Teramind, Securonix.
Device attestation
Zero-trust device posture checks that verify device identity, health, and network state before granting resource access. Blocks access from non-compliant or dual-homed devices. Zscaler, Cloudflare Access, Palo Alto Prisma.

The practitioner takeaway

If your detection strategy depends on seeing traffic on your network, you have a blind spot. Residential proxies make location-based detection unreliable. Personal hotspots make network-based detection irrelevant. Both are trivially easy for any employee to use, require no technical sophistication, and leave no obvious artifacts in your existing telemetry.

The shift is from network-centric monitoring to endpoint-centric + behavioral monitoring. This isn't a future state — it's a current gap. If you haven't tested your detection coverage against these scenarios, now is the time.
