Changelog

Version history for the Insider Threat Analyst Career Framework. This site is actively maintained and updated based on community feedback, new research, and evolving practice.

v1.3.1 2026-07-18
  • Changed ATT&CK mapping refreshed from v18.1 to v19. ATT&CK v19 (2026) renamed the Defense Evasion tactic to Stealth (TA0005) and introduced a new Defense Impairment tactic (TA0112); techniques that mapped cleanly were retagged to Stealth, and the restructured Impair Defenses family (revoked in v19) was set aside for manual remapping rather than guessed.
  • Changed De-duplicated the Insider Threat Matrix listing in References — the project canonicalizes to insiderthreatmatrix.org (the .com domain redirects there), so the duplicate .com entry was merged into the single Forscie entry.
  • Fixed ITMG entry updated to canonical URLs; IRCF name corrected to the Insider Risk Capability Framework (not “Control”) — with thanks to Shawn Thompson (ITMG).
  • New Shawn M. Thompson (ITMG founder) added to Contributors — canonical ITMG Body of Knowledge and Capability Framework links, the IRCF naming correction, and free e-book access for the 90-Day Plan.
v1.3.0 2026-07-18
  • New The site source is now public on GitHub — github.com/robknowles-cybersecurity/insiderthreatanalyst.com (MIT-licensed code, content under CC BY 4.0). Issues and pull requests welcome. A "Source on GitHub" link now appears in the footer across the site.
  • New Agentic AI deep-dive on the Emerging Threats page — autonomous agents as insiders ("superuser doppelganger"), the "ClawJacked" incident class (Oasis Security / OpenClaw), and 2026 exposure data (Gartner 40% agent adoption by year-end; Thales 61% rank AI as top data risk; DTEX/Ponemon 44% expect agent-driven theft vs 19% treating agents as insider-equivalent).
  • New CISA Assembling a Multi-Disciplinary Insider Threat Management Team (POEM lifecycle, Jan 2026) added to References and the Program governance section.
  • New CISA / NSA / Five Eyes Careful Adoption of Agentic AI Services (May 2026) added to the AI/ML references.
  • New Framework ecosystem additions to References — the Insider Threat Matrix (Forscie; Above Security named inaugural sponsor, Apr 2026), the ITMG® Insider Risk Body of Knowledge™ + IRCF v1.0, and MITRE CTID's 2026 project slate (Secure AI with ATLAS, Fight Financial Fraud, Ambiguous Techniques).
  • New The Human Risk Matrix (v0.1, humanriskmatrix.org) added to References as an early-stage open taxonomy of human risk mapped to MITRE ATT&CK.
  • New Practitioner reading — Thompson & Friedlander, Insider Threat Program: Your 90-Day Plan (ObserveIT, 2016).
  • Fixed KSA count corrected to 144. The framework explorer's data set contains 144 unique KSA ids (9 NICE, 103 custom, 24 adjacent, 8 tool/technique); current-state references that read "122" have been corrected. Historical changelog entries are left as they were shipped.
  • Fixed Concentration count corrected to 9 — the Program Auditor concentration (added in v1.1.0) was not reflected in the summary count.
  • Changed Insider-risk cost benchmark refreshed to Ponemon/DTEX 2026 — $19.5M average annual cost (up from $17.4M), 67-day containment, insider risk management ~19% of the security budget.
  • Changed Accessibility (WCAG 2.1 AA) — programmatic labels added to every form input on the framework explorer, suggestion form, and learning-path builder (1.3.1 / 4.1.2); prefers-reduced-motion support added site-wide; the muted-gray text tier adjusted to meet the 4.5:1 body-text contrast ratio in both themes (1.4.3).
  • Changed Hardening — Subresource Integrity (SRI) pinned on the external jsPDF library; a privacy disclosure was added for suggestion-form (Formspree) submissions.
  • Removed Pre-edit .bak source backups purged from the web root (and blocked at the server).

v1.1.3 2026-05-15
  • Improved Filter chips on the KSA matrix now have a per-group all-on / all-off toggle (the small ⇅ next to each filter label). Click toggles between turning every chip in the group on and turning every chip off — handy for the 19-chip Cluster group. Mixed states snap to all-on first.
  • Changed Matrix filter semantics: an empty filter group (zero chips selected) now means “no constraint from this dimension” rather than “match nothing.” So if you toggle every section off and then turn on just Knowledge in the Type group, you’ll see all Knowledge KSAs across every level, source, concentration, and cluster — standard faceted-search behavior. All-chips-on still shows everything; the only behavior change is around the all-off state.
  • Improved Section counts on the matrix (Knowledge, Skills, Abilities, Tasks) now update live as you filter, rather than showing the full-dataset count. Sections that drop to zero rows under the current filter are hidden entirely so you don’t see an empty table.
